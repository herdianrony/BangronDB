BangronDB
Fitur API Quick Start Release
SQLite document database untuk PHP 8.1+

API dokumen bergaya MongoDB, ringan, terenkripsi, dan tanpa server terpisah.

BangronDB memudahkan Anda membangun aplikasi PHP dengan penyimpanan berbasis SQLite, query fleksibel, searchable fields, hooks, schema validation, serta lifecycle API yang eksplisit untuk database dan collection.

Lihat Repository Baca Dokumentasi
PHP 8.1+ SQLite AES-256-GCM Non-lazy lifecycle API 
use BangronDB\Client;

$client = new Client(__DIR__ . '/data');
$client->createDB('app');
$client->createCollection('app', 'users');

$users = $client->selectCollection('app', 'users');

$id = $users->insert([
    'name' => 'John Doe',
    'email' => 'john@example.com',
    'role' => 'admin',
]);

$user = $users->findOne(['_id' => $id]);

Fitur utama

Dirancang untuk aplikasi PHP yang membutuhkan embedded data layer yang mudah dipakai, aman, dan cukup kaya fitur untuk use case nyata.

Document API

CRUD dokumen dengan pola query yang familier bagi pengguna MongoDB.

SQLite backend

Storage file-based atau in-memory tanpa perlu menjalankan DB server terpisah.

Encryption + searchable fields

Dokumen terenkripsi dengan dukungan field pencarian untuk use case autentikasi dan data sensitif.

Schema validation

Validasi type, enum, regex, min/max, dan aturan dasar lain sebelum data disimpan.

Hooks & soft deletes

Tambahkan lifecycle hooks, ubah alur proses, atau gunakan penghapusan logis dengan restore.

Metrics & monitoring

Integrity check, health report, collection metrics, dan informasi performa database.

Lifecycle API yang eksplisit

BangronDB sekarang memisahkan dengan jelas antara membuat resource dan memilih resource yang sudah ada.

Database
  • createDB()
  • dbExists()
  • selectDB()
  • renameDB()
  • dropDB()
Collection dari Client
  • createCollection()
  • collectionExists()
  • selectCollection()
  • renameCollection()
  • dropCollection()
Collection dari Database
  • createCollection()
  • collectionExists()
  • selectCollection()
  • renameCollection()
  • dropCollection()
Catatan migrasi: selectDB() dan selectCollection() sekarang non-lazy. Untuk resource baru, gunakan createDB() dan createCollection().

Mulai cepat

Instal library lalu gunakan lifecycle API eksplisit untuk membuat database dan collection.

Quick demo
Create explicitly, select safely
use BangronDB\Client;

// 1. create resources explicitly
$client = new Client(__DIR__ . '/data');
$client->createDB('app');
$client->createCollection('app', 'users');

// 2. select existing resources
$users = $client->selectCollection('app', 'users');

$id = $users->insert([
  'name' => 'John Doe',
  'email' => 'john@example.com',
  'role' => 'admin',
]);

$user = $users->findOne(['_id' => $id]);
Migration note: selectDB() dan selectCollection() kini non-lazy. Gunakan createDB() dan createCollection() untuk membuat resource baru.

Fitur utama

Dirancang untuk use case embedded data layer yang cepat dipakai, aman, dan cukup kaya fitur untuk aplikasi nyata.

MongoDB-like document API

CRUD, query operators, sorting, projection, pagination, dan save/upsert dengan sintaks yang familier.

SQLite backend

File-based atau in-memory, cocok untuk aplikasi lokal, tools internal, demo, atau embedded workflows.

Encryption + searchable fields

Dokumen terenkripsi dengan field pencarian untuk autentikasi dan data sensitif tanpa mengorbankan query dasar.

Schema validation

Lindungi data lebih awal dengan validasi type, enum, regex, min/max, dan aturan dasar lainnya.

Hooks & soft deletes

Tambahkan lifecycle hooks, veto operation, audit metadata, atau penghapusan logis dengan restore.

Metrics & monitoring

Integrity check, health report, collection metrics, dan informasi performa database siap dipakai.

Lifecycle API yang eksplisit

BangronDB kini memisahkan dengan jelas antara membuat resource baru dan memilih resource yang sudah ada.

Database
  • createDB()
  • dbExists()
  • selectDB()
  • renameDB()
  • dropDB()
Collection dari Client
  • createCollection()
  • collectionExists()
  • selectCollection()
  • renameCollection()
  • dropCollection()
Collection dari Database
  • createCollection()
  • collectionExists()
  • selectCollection()
  • renameCollection()
  • dropCollection()

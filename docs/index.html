Database dokumen
untuk PHP, tanpa ribet.

Saya bikin BangronDB karena butuh database yang ringan — MongoDB-style API di atas SQLite, tanpa server, tanpa config. Cocok untuk aplikasi yang di-deploy per-customer atau embedded di appliance.

Features

Fitur lengkap,
tapi tetap ringan

Saya cuma masukin fitur yang benar-benar dipakai di production. Tidak ada bloat — schema validation, encryption, hooks, aggregation, semua optional dan composable.

MongoDB-Style API

Familiar find(), insert(), update(), aggregate() syntax. No learning curve.

Dual Query Strategy

SQL-first via json_extract for speed, PHP fallback for complex queries. Automatic.

AES-256-GCM Encryption

Per-collection encryption with key rotation, searchable encrypted fields via HMAC blind index.

Schema Validation

Type checking, enum, regex, min/max, unique constraints. Metadata properties for UI generation.

Hook System

beforeInsert, beforeUpdate, beforeRemove hooks. Build ACL, audit logs, auto-timestamps at app layer.

Zero Dependencies

Only PHP 8.1+ with pdo_sqlite and openssl. No MongoDB, no Redis, no external services.

Architecture

Arsitektur berbasis Trait

Setiap capability (encryption, hooks, soft-delete, dll) adalah trait terpisah. Class Collection tinggal `use` trait yang dibutuhkan — horizontal reuse, bukan inheritance rumit.

Collection.php (Coordinator)
QueryBuilderTraitfind(), findOne(), count(), aggregate()
EncryptionTraitAES-256-GCM, key rotation
SchemaValidationTraittype, enum, regex, min/max, unique
HooksTraitbefore/after CRUD lifecycle
SearchableFieldsTraitHMAC blind index for encrypted data
IdGeneratorTraitUUID, manual, prefix
SoftDeleteTraitsoft delete, restore
TtlTraitauto-expiration
ChangeTrackingTraitversioning, notifications
ConfigurationPersistenceTraitpersistent config
💡 Traits prevent God Object. Each feature is an independent, testable unit.
